# work_260401

## 우선순위 1: 기본 admin seed 제거

- 문제: 초기 마이그레이션에서 `admin/admin` 기본 계정이 자동 생성된다.
- 영향: 운영 환경에서 기본 자격 증명이 노출되면 즉시 계정 탈취로 이어질 수 있다.
- 권장 작업: 기본 관리자 insert를 제거하고, 초기 관리자 생성은 별도 bootstrap 절차나 수동 운영 절차로 분리한다.
- 완료 기준: 신규 DB를 띄워도 기본 관리자 계정이 자동으로 생성되지 않으며, 문서에 초기 관리자 생성 방식이 명시된다.

## 우선순위 2: `APP_JWT_SECRET` 필수화 및 문서/env 반영

- 문제: JWT secret이 누락되면 개발용 기본값으로 기동된다.
- 영향: 운영에서 secret 설정이 빠져도 서버가 떠서, 토큰 위조 위험이 생긴다.
- 권장 작업: `APP_JWT_SECRET`이 없으면 애플리케이션이 시작되지 않도록 변경하고, `README.md`와 `llm.env.example`에 필수 항목으로 반영한다.
- 완료 기준: secret 미설정 시 부팅 실패가 발생하고, 배포 문서와 예제 env에 설정 방법이 포함된다.

## 우선순위 3: AI 호출 timeout/retry 추가

- 문제: 외부 AI API 호출에 명시적 timeout과 재시도 전략이 없다.
- 영향: 공급자 지연이나 일시 장애 시 요청 스레드가 오래 묶이고, 관리자 UI가 멈춘 것처럼 보일 수 있다.
- 권장 작업: OpenAI/Claude/Grok 호출에 connect/read timeout을 설정하고, 제한된 retry 정책과 실패 메시지 정리를 추가한다.
- 완료 기준: AI API 지연 상황에서 무한 대기가 발생하지 않고, 실패가 정해진 시간 안에 사용자에게 반환된다.

## 우선순위 4: `localStorage` 토큰 저장 방식 개선

- 문제: 관리자 JWT를 `localStorage`에 저장하고 있다.
- 영향: XSS가 발생하면 토큰이 직접 탈취될 수 있다.
- 권장 작업: 프론트의 인증 방식을 `HttpOnly` 쿠키 기반 세션으로 전환하는 방향으로 설계하고, 백엔드는 쿠키 인증/갱신 흐름을 지원하도록 정리한다.
- 완료 기준: 브라우저 JS가 토큰 원문을 직접 보관하지 않으며, 인증은 쿠키 기반으로 동작한다.

## 진행 메모

- 이 4개는 기능 개선보다 운영 안정성과 보안에 직접 영향을 준다.
- 구현 순서는 `1 -> 2 -> 3 -> 4`로 두는 것이 안전하다.
